Turn Workflow

End-to-end sequence of what happens when a user utterance arrives. Entry point: InterviewEngine.process_turn(user_input).

Interview phase transitions

_update_phase() runs at the top of every turn:

stateDiagram-v2
    [*] --> OPENING: turn 1
    OPENING --> CORE: turn > 2
    CORE --> DEEP_DIVE: chain_depth >= 2
    CORE --> DEEP_DIVE: kg_gap_score < 0.3
    DEEP_DIVE --> CORE: chain_depth resets
    CORE --> CLOSING: turn >= 25
    DEEP_DIVE --> CLOSING: turn >= 25
    CLOSING --> [*]

Figure 7 — Phase state machine. Phase feeds into router's complexity-score bias.

Phase is a routing signal:

PhaseComplexity biasEffect
OPENING−0.20Push toward S1 — rapport building
CORE0.00No bias
DEEP_DIVE+0.20Push toward S2 — thorough probing
CLOSING−0.15Back to S1 — wrap up

Streaming contract

process_turn returns an AsyncIterator[StreamEvent]. Callers consume it like this (from demo.py):

async for event in engine.process_turn(user_input):
    if isinstance(event, TextDelta):
        print(event.text, end="", flush=True)
    elif isinstance(event, StreamDone):
        # per-turn metrics
        print(f"TTFT: {event.latency.ttft_ms}ms")
    elif isinstance(event, StreamError):
        print(f"Error: {event.error}")

Three event types reach the caller:

EventWhenContains
TextDeltaThroughout streamA chunk of visible response
StreamDoneOnce per turnusage, latency, stop_reason
StreamErrorOn API failureerror string

ThinkingDelta events are not yielded to the caller — they're consumed inside system2.generate and speculative.generate_speculative.

Failure modes

What failsWhat happens
Anthropic API error mid-streamStreamError yielded; partial response (if any) still recorded
Follow-up classification failsHeuristic fallback in followup._heuristic_classify()
Speculative draft missing confidenceDefaults to 0.5 → triggers verify path
Speculative S2 verify failsFalls back to shipping the draft as-is
Persona file missingswitch_persona() returns False, keeps current persona
KG extraction throwsLogged; turn continues with stale KG summary

The engine is built to always produce a turn record even on failure, so session analytics stay consistent.
