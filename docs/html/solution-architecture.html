3. Module responsibilities

Orchestration — reason_sot/interview/

Everything that ties per-turn state together.

FileRole
engine.py — InterviewEngine Top-level orchestrator. process_turn(user_input) is the only entry point callers need. Holds per-session state: conversation history, phase, follow-up chain depth, persona, KG.
followup.py Classifies candidate response into NEXT_TOPIC / CLARIFY / EXPLORE using a Haiku call with JSON output. Heuristic fallback if the LLM call fails.
knowledge_graph.py In-memory graph of topics, entities, skills, experiences. Updated each turn via regex extraction. Serializes to a summary string for the cached system block.

Reasoning — reason_sot/core/

Pure reasoning logic. No session state, no persona logic — these modules take messages in and yield stream events out.

FileRole
router.pyDeterministic routing: user input → RoutingDecision. No LLM call.
system1.pyThin wrapper: Haiku, no thinking, 512 max tokens.
system2.pySonnet with extended thinking. Injects CoT/MoT/DST instruction into the last user message (not system prompt — that would bust cache).
mot.pyMatrix-of-Thought prompt builder. MoTConfig.from_signals(...) picks broad/deep/balanced.
dst.pyDomain-Specialized Tree prompt builder with adaptive beam logic.
speculative.pyS1 draft → confidence gate → optional S2 verify. Combines usage and latency from both calls.
early_exit.pyPre-call thinking-budget sizing; post-call analysis for adaptive multiplier on next turn.

LLM boundary — reason_sot/llm/

Everything that talks to Anthropic goes through here.

FileRole
client.py — LLMClientAsync streaming wrapper. Yields StreamEvent. Forces temperature=1.0 when thinking is enabled.
cache.pyBuilds the 4-block cache-controlled system array and marks the last user message.
models.pyModel registry: ModelTier.FAST / DEEP → ModelSpec. Override via override_models().

Persona — reason_sot/persona/

Drives what the interviewer is — their role, domain, topics to cover, decision patterns.

FileRole
profiles.pyPydantic models: PersonaProfile, TopicArea (priority 1=MUST / 2=SHOULD / 3=OPTIONAL), DecisionPattern.
manager.pyLoads YAML, renders [base_system, persona_prompt] for the cache builder, suggests persona switches.
definitions/*.yamlShipped personas: technical_interviewer, behavioral_interviewer, career_coach.

Scoring — reason_sot/scoring/

Post-hoc analysis. Not on the hot path.

FileRole
metrics.pyFour quality scores (depth / breadth / persona / follow-up relevance), weighted into an overall score.
latency.pyTTFT / total-duration / cache-hit-rate profiler.
benchmarks.pyComparative harness: ReasonSoT vs. S1-only vs. S2-always.

6. Interview phases

The phase state machine drives routing signals. _update_phase() runs at the top of every turn and reads turn_count, followup_chain_depth, and KG coverage gap score.

    OPENING --> CORE: turn > 2

7. Extension points

The design assumes you might want to extend or replace these:

I want to…Touch this
Add a new reasoning mode core/ — add mymode.py with build_mymode_prompt(); extend ReasoningMode enum in types.py; add a case in system2._augment_messages_for_mode()
Change routing behavior core/router.py — edit _compute_complexity_score() or the pattern lists at the top
Add a persona Drop a YAML into reason_sot/persona/definitions/
Change cache block placement llm/cache.py::build_system_blocks() — be careful, this affects hit rates
Swap LLM provider Replace llm/client.py — the rest of the codebase only depends on its public methods and StreamEvent types
Add a quality metric scoring/metrics.py — add _score_myscore() and wire into score_session()

8. Design constraints

Things that must stay true

  • Router is stateless and LLM-free. Introducing an LLM call here would reintroduce the latency we're trying to avoid.
  • Reasoning-mode instructions go in the last user message, not the system prompt. System-prompt changes bust the prefix cache (block 2 onward).
  • Thinking deltas never reach the caller. system2.generate and speculative.generate_speculative consume them internally.
  • Cache block order is fixed: base → persona → KG → last user. Changing this invalidates the cache prefix for everything downstream.
  • process_turn always produces an InterviewTurn record, even on failure, so session analytics stay consistent.

Things we deliberately don't do

  • No tool use, no agent loop, no external API integrations.
  • No fine-tuning. All behavior comes from prompts, routing, and cache placement.
  • No cross-session memory. Each InterviewEngine starts with an empty KG.
  • No vector embeddings in the KG — keyword matching only. Speed over sophistication.
  • No retry logic for Anthropic API errors. Failures become StreamError events; callers decide.
